Namespace: Eco.XComp.Factory

Eco.XComp.Factory

Factory.js파일 at nexacro® software. Eco.XComp.Factory Namespace 정의
XComp를 application running시에 동적으로 생성을 필요시에 cache 처리하여 매번 create하지 않고
처리되는 기능을 담고 있는 함수들의 집합 Namespace이다.

Eco.XComp.Factory 생성된 컴포넌트들 관리 방법

주어지는 form 컴포넌트에 사용자 속성으로 [_used], [_cache] 명칭의 simple object가 존재하게 된다.

pool명칭을 지정하지 않은 경우
-----------------------------------------------
_defaultUsed : 사용중인 _id 값으로 이루어진 object collection
_defaultCache : [comp Type명칭] 속성으로 사용하지 않은 comp를 담고 있는 array

pool명칭을 지정한 경우
-----------------------------------------------
_used : [pool명칭] 속성으로 사용중인 _id 값으로 이루어진 object collection
_cache : [pool명칭] 속성으로 사용하지 않은 comp를 담고 있는 array

또한 XComp 속성 값 할당 및 position, size 변경 처리를
XComp에 바로 반영하지 않고 예약 기록하여 필요시 한번에 처리하여
불필요한 화면 render 줄이는 기능을 담고 있다.

release

사용하고 있는 XComp인 주어진 c에 대하여 cache처리한다.
doLayout 함수 호출 전까지는 실질로 화면상에 보여진다.
하지만 화면상에 반영되는 것과 별개로 cache된 기록은 내부적으로 처리된다.
두번째 인자는 임의 poolName에 대하여 처리하고자 할 때 설정한다. 세번째 param은 cache처리할 때 기 설정된 속성값들을 clear 할 것인지 여부이다.

releaseAll

주어진 p 즉 form 객체에 속한 사용중인 comp를 모두 cache 처리한다.
doLayout 함수 호출 전까지는 실질로 화면상에 보여진다.
하지만 화면상에 반영되는 것과 별개로 cache된 기록은 내부적으로 처리된다.
두번째 param인 exclude는 XComp array인데 이것은 cache처리할 대상에서 제외한다.
세번째 param은 cache처리할 때 기 설정된 속성값들을 clear 할 것인지 여부이다. 첫번째 인자는 form이거나, object로 {target: form객체, poolName: "rowobjs"} 구성 될 수 있다. object로 주어질때는 poolName를 임의로 지정하여 사용하는 경우이다.

releaseNamed

주어진 p(form|div|tabpage) 객체에 속하고, 주어진 명칭 값을 담고 array에 해당하는 comp를 모두 cache 처리한다.
doLayout 함수 호출 전까지는 실질로 화면상에 보여진다.
하지만 화면상에 반영되는 것과 별개로 cache된 기록은 내부적으로 처리된다.
첫번째 인자는 form이거나, object로 {target: form객체, poolName: "rowobjs"} 구성 될 수 있다. object로 주어질때는 poolName를 임의로 지정하여 사용하는 경우이다.

setProperties

주어진 c 즉 XComp의 속성 값을 설정한다.
Eco.XCompFactory.doLayout 함수를 호출하기 전까지 실질로 화면상에 변경이 발생하지 않는다. 설정된 정보를 _props에 가지고 있다가 doLayout함수 호출하는 시점에 반영하고 _props정보는 clear된다.

setRect

주어진 c 즉 XComp의 rect 정보를 설정한다.
